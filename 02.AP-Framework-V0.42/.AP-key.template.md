# AP-Key: 프로젝트 서비스 키 관리

> 이 파일에는 프로젝트에서 사용하는 서비스 URL과 인증 키를 기록합니다.
> 교육 시작 전 또는 1주차에 모든 항목을 채워주세요.
>
> **주의**: 이 파일에는 민감한 인증 정보가 포함됩니다.
> 공개 저장소에 Push하지 않도록 `.gitignore`에 추가하세요.

---

## 1. GitHub

| 항목 | 값 |
|------|-----|
| 저장소 URL | `https://github.com/{username}/{repo-name}` |
| Personal Access Token (PAT) | `ghp_xxxxxxxxxxxxxxxxxxxx` |
| PAT 권한 | repo, workflow, project |
| PAT 만료일 | YYYY-MM-DD |

> PAT 발급: GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic)
> `project` 권한은 GitHub Projects 보드 연동에 필요합니다.

---

## 2. Slack

| 항목 | 값 |
|------|-----|
| 워크스페이스 URL | `https://{workspace}.slack.com` |
| 알림 채널 | #{채널명} |
| Incoming Webhook URL | `https://hooks.slack.com/services/T.../B.../xxx` |

> Webhook 설정: Slack > Apps > Incoming Webhooks > Add to Slack > 채널 선택
> Slack Incoming Webhook URL은 n8n Credential이 아닙니다. 워크플로우 내 HTTP Request 노드에 직접 입력합니다.

## 4. n8n

> 워크플로우 JSON은 Claude Code가 생성하고, PM이 n8n Cloud에 import + Credential 연결만 수행한다.

| 항목 | 값 |
|------|-----|
| n8n Cloud URL | `https://{instance}.app.n8n.cloud` |

**워크플로우 Webhook**:

| # | 워크플로우 | 용도 | Webhook 경로 |
|---|-----------|------|-------------|
| 1 | GitHub Issue -> Slack 알림 | Issue 생성/변경 시 알림 | /webhook/github-issue-webhook |
| 2 | 주간 요약 -> Slack 전송 | 매주 금요일 진척 요약 | /webhook/weekly-summary |
| 3 | 배포 알림 -> Slack 전송 | CI/CD 완료 시 결과 알림 | /webhook/deploy-notification |
| 4 | Slack AI Agent (Chat UI) | n8n Chat에서 프로젝트 현황 질의 | (Chat Trigger) |
| 5 | Slack PM Agent | Slack 슬래시 명령으로 현황 질의 | /webhook/slack-pm-agent |

**n8n Credentials**:

| Credential | 등록할 키 | 상태 |
|-----------|----------|------|
| GitHub API | PAT (1번 항목) | [ ] |
| Google Gemini API | API Key (WF 04, 05용) | [ ] |

> Slack Incoming Webhook URL은 Credential이 아님. 워크플로우 내 HTTP Request 노드에 직접 입력.
