## 1. 기능 요구사항 (Functional Requirements)

> ID 체계: `REQ-{기능군}-{순번}` — F1~F5는 MVP, P2/P3는 Phase별, OPS는 운영 공통

### 1.1 F1. 위치 기반 식당 탐색

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|----|------|----------|----------|-----------|
| REQ-F1-001 | 탐색 | 사용자 현재 위치를 중심으로 식당을 지도 뷰로 표시 | **Must** | GPS 권한 허용 시 1초 이내 현재 위치 핀 + 주변 식당 마커 표시. 권한 거부 시 수동 위치 검색 폴백 |

| REQ-F3-006 | 쿠폰 | **1일 사용 횟수 제한** (사용자당 일 5건 / 매장당 사용자 1건) | **Must** | 양도·도용 방지를 위한 정책. 초과 시 사용 차단 |
| REQ-F3-007 | 쿠폰 | 쿠폰 만료 관리 (당일 만료 / 7일 만료 / 사장님 지정 만료) | **Must** | 만료 1시간 전 푸시 알림 (옵트인 사용자). 만료 후 자동 비활성 |

### 1.4 F4. 사장님 매장·특선 관리

| ID | 구분 | 요구사항 | 우선순위 | 인수 기준 |
|----|------|----------|----------|-----------|
| REQ-F4-001 | 사장님 | **사업자 인증** (휴대폰 본인 인증 + 사업자 등록번호 + 국세청 진위 확인) | **Must** | 미인증 사장님은 매장 등록 불가. 인증 완료 평균 5분 이내 |
| REQ-F4-002 | 사장님 | 매장 등록 — 매장명, 카테고리, 주소(지도 좌표), 운영시간, 전화번호, 대표 사진 | **Must** | **3단계 이내** 완료 (1: 기본정보, 2: 운영시간·메뉴, 3: 사진·확인). 평균 5분 이내 |
| REQ-F4-003 | 사장님 | 매장 등록 시 메타데이터 필수 입력 — `solo_seating` (1인 좌석), `seats` (좌석 수), `meal_duration` (15~25/25~40/40+분) | **Must** | 페르소나 차별화 핵심 메타. 미입력 시 등록 완료 불가 |
| REQ-F4-004 | 사장님 | **일별 점심특선 등록** — 메뉴명, 정상가, 할인가(또는 할인율), 한정 표시 여부, 사진 | **Must** | 등록 후 즉시 사용자 피드 노출. 등록·수정 평균 30초 이내 |
| REQ-F4-005 | 사장님 | **요일별 점심특선** 등록 (예: 매주 월·수 11~14시 자동 노출) | **Must** | 요일 + 시간대 반복 설정. 휴무일·임시 휴업 토글 별도 |
| REQ-F4-006 | 사장님 | 즉시 발행 / 예약 발행 (특정 일시에 자동 노출) | **Must** | 예약은 분 단위 정확도. 발행 후 취소 가능 (사용 전 한정) |
| REQ-F4-007 | 사장님 | 실시간 영업 상태 임시 토글 (영업중 / 잠시 중단 / 마감) | **Must** | 토글 즉시 사용자 피드 반영. "잠시 중단"은 자동 30분 후 복귀 |
| REQ-F4-008 | 사장님 | 매장 정보·메뉴 수정 (사진·가격·운영시간 등) | **Must** | 수정 이력 30일 보관 (분쟁 대비) |

| REQ-F5-008 | 대시보드 | 통계 데이터 CSV/Excel 내보내기 | **Should** | 월 단위 일괄 내보내기. 매장 회계용 |
