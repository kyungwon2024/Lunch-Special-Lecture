## 6가지 제품 우선순위 프레임워크 (일반 요약)

### 1. RICE Scoring
- **공식**: (Reach × Impact × Confidence) / Effort
- **장점**: 정량적, 비교 가능
- **단점**: 추정값 의존도가 높음
- **적합**: 다수 기능 후보에서 합리적 선택 필요 시

### 2. Kano Model
- 기능을 5단계로 분류: Must-be / Performance / Attractive / Indifferent / Reverse
- **장점**: 고객 만족도와 직결
- **단점**: 사용자 조사 필요
- **적합**: 신규 제품의 차별화 기능 결정

### 3. MoSCoW (Must / Should / Could / Won't)
- 점심특강 PRD에 이미 적용 중
- **장점**: 단순·빠른 합의
- **단점**: 정량적 비교 어려움
- **적합**: 스프린트/릴리즈 범위 결정

### 4. Value vs Effort Matrix (2x2)
- Quick Wins / Big Bets / Maybes / Time Sinks
- **장점**: 시각적, 직관적
- **단점**: 두 축으로만 평가
- **적합**: 워크숍, 빠른 우선순위 합의

### 5. Cost of Delay (CoD)
- 기능 지연 시 발생하는 비용을 산출
- **장점**: 경영진 설득에 효과적
- **단점**: 화폐 가치 산정 어려움
- **적합**: B2B, 시간 민감 시장

### 6. Story Mapping
- 사용자 여정을 가로축, 우선순위를 세로축으로 매핑
- **장점**: 전체 사용자 경험 가시화
- **단점**: 시간 소요
- **적합**: 신규 제품, 큰 릴리즈 계획

## 프레임워크 선택 가이드

| 상황 | 권장 프레임워크 |
|------|----------------|
| 합리적 정량 비교 | RICE |
| 고객 만족도 차별화 | Kano |
| 빠른 합의 | MoSCoW |
| 워크숍/팀 정렬 | Value vs Effort |
| 경영진 설득 | Cost of Delay |
| 신규 제품 전체 설계 | Story Mapping |

## 점심특강 프로젝트 활용 시사점

- 현재 **MoSCoW** 적용 중 (PRD.md 부록 C)
- MVP 출시 후 Phase 2 기능 우선순위는 **RICE** 추천 (정량 데이터 축적 후)
- 사장님 앱 차별화 기능은 **Kano Model**로 사용자 조사 후 결정
