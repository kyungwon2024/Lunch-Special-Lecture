# 프롬프트 라이브러리

이 폴더에는 주차별로 사용하는 Claude Code 프롬프트가 정리되어 있습니다.

## 사용 방법

1. 각 주차 파일에서 필요한 프롬프트를 찾습니다
2. 터미널에서 `claude "프롬프트 내용"` 형태로 실행합니다
3. 생성된 산출물을 검토하고 필요 시 수정을 요청합니다

## 주차별 프롬프트 목록

| 파일 | 주차 | 프롬프트 수 | 용도 |
|------|------|-------------|------|
| week1-초기화.md | Week 1 | P-101 ~ P-104 | 프로젝트 초기화, PRD 작성, 통합자료실 셋업, NLM 동기화 |
| week2-관리자동화.md | Week 2 | P-201 ~ P-208 | 착수보고서, WBS, Issues, 주간보고서, 중간/완료보고서 |
| week3-기획자동화.md | Week 3 | P-301 ~ P-309 | 마켓리서치~화면설계서, Group A 병렬 실행 |
| week4-구현자동화.md | Week 4 | P-401 ~ P-409 | 프론트엔드, 백엔드, 배포, 결제, Group B 병렬 실행 |
| week5-검수자동화.md | Week 5 | P-501 ~ P-506 | 테스트, CI/CD, 결과 보고서, 배포 검증 |

## 공통 프롬프트

| ID | 프롬프트명 | 용도 |
|----|-----------|------|
| P-001 | CLAUDE.md 템플릿 | 프로젝트 규칙, 기술 스택, 문서 형식 등 Claude Code 기본 행동 설정 |
| P-002 | PM 시스템 프롬프트 (범용) | Claude 웹에서 PM 역할 설정 |
| P-003 | AI 결과물 검증 체크리스트 | 생성된 산출물/코드의 정확성/완전성 검증 |

## 프롬프트 작성 원칙

- 이전 산출물을 명시적으로 참조합니다 (예: "요구사항정의서.md를 기반으로")
- 원하는 출력 형식을 구체적으로 지정합니다 (예: "마크다운 표로 출력")
- CLAUDE.md에 기본 규칙이 설정되어 있으므로, 프롬프트에서는 해당 작업에 고유한 지시만 포함합니다
