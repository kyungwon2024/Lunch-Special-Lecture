# Week 3 프롬프트: 기획 자동화

## P-301. 마켓리서치 생성기

```
claude "PRD.md의 프로젝트 개요를 기반으로 마켓리서치를 수행해줘. 타겟 시장, 경쟁사, 기술 동향, 사용자 니즈를 조사하고 리서치 자료 10건 이상을 정리해줘"
```

**입력**: PRD.md
**출력**: 02.기획문서/마켓리서치.md
**참고**: 웹 검색을 활용하여 실제 데이터 기반의 리서치를 수행합니다. 이 결과가 서비스기획서의 입력이 됩니다.

---

## P-302. 서비스기획서 생성기

```
claude "마켓리서치.md를 기반으로 서비스기획서를 작성해줘. 타겟 사용자, 서비스 전략, 차별화 포인트, 브랜드 전략, 수익 모델, 개발 전략을 포함해줘"
```

**입력**: 02.기획문서/마켓리서치.md, PRD.md
**출력**: 02.기획문서/서비스기획서.md
**참고**: 마켓리서치 데이터를 인용하여 근거 기반의 기획서를 작성합니다. 이 결과가 요구사항정의서의 입력이 됩니다.

---

## P-303. 요구사항 정의서 생성기

```
claude "서비스기획서.md와 착수보고서.md를 분석하여 요구사항정의서를 작성해줘. 형식: ID/구분/요구사항/우선순위/인수기준. 마크다운 표로 출력"
```

**입력**: 02.기획문서/서비스기획서.md, 01.관리문서/착수보고서.md
**출력**: 02.기획문서/요구사항정의서.md
**참고**: MoSCoW (Must/Should/Could/Won't) 우선순위를 적용합니다

---

## P-304. 기능 명세서 생성기

```
claude "요구사항정의서.md를 기반으로 기능명세서를 작성해줘. 각 기능별 입력/처리/출력/예외처리를 포함해줘"
```

**입력**: 02.기획문서/요구사항정의서.md
**출력**: 02.기획문서/기능명세서.md
**참고**: 요구사항 ID와 기능 ID의 매핑을 명확히 합니다

---

## P-305. API 스펙 생성기

```
claude "기능명세서.md를 기반으로 RESTful API 스펙을 작성해줘. 엔드포인트별 Method, URL, Request, Response, 에러코드를 포함해줘"
```

**입력**: 02.기획문서/기능명세서.md

## P-309. Group A 병렬 실행 (기능명세서 이후)

> V0.3 신규. 기능명세서(#6) 완료 후 독립적인 4개 산출물을 동시 생성합니다.

```
"기능명세서(#6)가 완료되었으니, 다음 4개 산출물을 병렬로 생성해줘.

1. API스펙 (#7): 기능명세서.md 기반 RESTful API 설계
2. 정보구조도 (#8): 기능명세서.md 기반 Mermaid flowchart
3. 시스템정의서 (#11): 기능명세서.md + 기술 스택 기반
4. 테스트시나리오 (#15): 기능명세서.md 기반 테스트 케이스

각 산출물은 독립적으로 생성하되, 기능명세서.md를 공통 입력으로 참조해.
완료 후 .progress.md를 일괄 업데이트해줘."
```

**입력**: 02.기획문서/기능명세서.md
**출력**: #7, #8, #11, #15 (4개 산출물 동시 생성)
**전제조건**: .progress.md에서 #6 상태=완료 확인
**실행 방식**: Claude Code Task 도구로 4개 서브에이전트 병렬 실행
