## 참고: 화면 설계 작성 팁

1. **플랫폼별 해상도**: 모바일 375px, PC 1920px 기준으로 설계
2. **ASCII 와이어프레임**: 코드 블록(```)으로 감싸서 레이아웃 표현
3. **컴포넌트 테이블**: 각 UI 요소의 타입, 설명, 인터랙션을 명확히 기술
4. **상태 정의**: 기본/로딩/에러/빈 상태를 반드시 고려
5. **접근 권한**: 화면별 접근 가능 사용자 역할 명시
6. **반응형**: 모바일/PC 모두 지원하는 경우, 각 해상도별 레이아웃 별도 작성
7. **네이밍 규칙**: 화면 ID는 `MOB-`, `WEB-`, `ADM-` 접두사로 플랫폼 구분
