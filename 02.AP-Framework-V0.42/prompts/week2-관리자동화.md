# Week 2 프롬프트: 관리 자동화

## P-201. 착수보고서 생성기

```
claude "README.md를 분석하여 착수보고서를 작성해줘. 프로젝트 개요, 목표, 범위, 일정, 팀 구성, 리스크를 포함해줘"
```

**입력**: README.md
**출력**: 01.관리문서/착수보고서.md
**참고**: 착수보고서 템플릿의 모든 섹션을 채워줍니다

---

## P-202. WBS 생성기

```
claude "착수보고서.md를 기반으로 WBS를 Level 3까지 생성해줘. 산출물, 담당자, 예상 공수를 포함해줘"
```

**입력**: 01.관리문서/착수보고서.md
**출력**: 01.관리문서/WBS.md
**참고**: 기획-설계-개발-테스트-배포 단계로 구분하여 작성합니다

---

## P-203. WBS -> Issues 변환기

```
claude "WBS.md의 각 태스크를 gh issue create 명령으로 GitHub Issue로 직접 생성해줘"
```

**입력**: 01.관리문서/WBS.md
**출력**: GitHub Issues (직접 생성)
**참고**: Claude Code가 `gh issue create` 명령을 실행하여 이슈를 직접 생성합니다. GitHub CLI(gh) 인증이 필요합니다.

---

## P-204. 주간 보고서 생성기

```
claude "GitHub Issues 현황을 분석하여 주간보고서를 작성해줘. 금주 실적, 이슈, 차주 계획을 포함해줘"
```

**입력**: GitHub Issues 현황 (gh issue list)
**출력**: 01.관리문서/주간보고서.md
**참고**: `gh issue list` 결과를 기반으로 진척률을 자동 계산합니다

---

## P-205. 경영진 보고서 변환기

```
claude "주간보고서를 경영진용 1-page 요약으로 변환해줘. 핵심 지표와 의사결정 포인트만 남겨줘"
```

**입력**: 01.관리문서/주간보고서.md
**출력**: 경영진 보고서 (1페이지 요약)
**참고**: 기술 디테일은 제거하고, 비즈니스 관점 요약에 집중합니다

## P-206. 리스크 등록부 생성기

```
claude "프로젝트 현황을 분석하여 리스크 등록부를 작성해줘. 영향도, 발생확률, 대응방안을 포함해줘"
```

**입력**: 01.관리문서/착수보고서.md, GitHub Issues 현황
**출력**: 리스크 등록부 (착수보고서 리스크 섹션 업데이트)
**참고**: 기존 착수보고서의 리스크 섹션을 최신 정보로 갱신합니다

## P-207. 중간보고서 생성기

```
claude "착수보고서 계획 대비 현재 진척 현황을 분석하여 중간보고서를 작성해줘. WBS 진척률, 산출물 현황, 이슈, 리스크, 향후 계획을 포함해줘"
```

**입력**: 01.관리문서/착수보고서.md, 01.관리문서/WBS.md, GitHub Issues 현황
**출력**: 01.관리문서/중간보고서.md
**참고**: 프로젝트 중간 시점(W3-4)에 작성합니다. 계획 대비 실적을 정량적으로 비교합니다.
